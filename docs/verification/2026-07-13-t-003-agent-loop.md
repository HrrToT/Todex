## 范围

本记录核验 T-003 的 Scripted Mock LLM、按 Run 编号的内存 TraceStore 与最小 `AgentRunner`。验证在独立 worktree `D:\Todex\.worktrees\t-003-agent-loop` 执行；不接入真实 LLM、网络、Electron、文件读写、Guardrail 或后续任务功能。

## 已核验行为

- 每个原始 LLM 输出先调用 `parseAction`，未知工具、缺失 `tool`、非对象输入均不会到达 Dispatcher。
- `read_file -> finish` 可确定性完成，`finish` 不调度 Dispatcher。
- ToolResult 被传入下一轮 `LlmTurnContext.previousResults`；`previousResults` 是每轮快照，历史 context 不受后续数组追加影响。
- TraceStore 为每个 Run 从 `0` 起连续编号，并按 Run 隔离读取。
- maxSteps、Mock 脚本耗尽、取消与 LLM/Dispatcher 终端错误均产生确定停止状态和 trace。
- Dispatcher 抛异常会转化为 `failed` ToolResult，回灌给下一轮 LLM，而不会从 `run()` 逃逸。

## 审查结论

初始实现经审查发现 4 个 P1：`previousResults` 使用可变数组引用；maxSteps 与脚本耗尽测试没有断言完整 trace/dispatcher 副作用；Dispatcher 异常路径无测试。`f57dad1` 在不改变 contracts 或外部依赖的前提下关闭了四项：上下文改为 `previousResults: [...results]`，并新增或增强相应断言。

仍有两个不阻塞 T-003 验收的 P2：`cancelledRuns` 对相同 Run ID 的复用语义尚未定义，TraceStore/Mock LLM 的公开观察对象尚未深度冻结。它们应在后续 Core 扩展前决定策略并补回归测试。

## 构建说明

PR #2 的首次 GitHub Actions 运行失败：全新 Ubuntu checkout 没有被 `.gitignore` 排除的 `packages/contracts/dist`，但 `@todex/contracts` 的 `main`、`types` 和 `exports` 都指向该目录；T-003 引入 workspace 依赖后，Vite 因而无法解析该包入口。本机先前残留的 `dist` 掩盖了此问题。

修复为给 contracts 增加 `tsconfig.build.json` 与 `build` 脚本，并将根及 Harness 的 `test` 改为先执行 `pnpm --filter @todex/contracts build`。已通过“临时移开 dist 后运行根和 Harness 测试”的红绿复验，且根 `build` 现在真实生成 contracts 产物。Electron 打包证据仍属于后续桌面端任务。
