## Verified invariants

- Raw LLM output is validated by `parseAction`; malformed input cannot reach governance or Dispatcher.
- `GovernanceController.evaluate` runs after parsing and before every live Dispatcher path. Hard denial and awaiting approval both yield zero dispatches.
- Workspace escape, symlink escape, sensitive credentials, complex shell composition, encoded/dynamic PowerShell, elevation, and system configuration are hard denials.
- `once`, `run`, `command_prefix`, and `deny` obey their defined scopes. Prefix grants do not cross projects, bypass compound shell input, or cover unsafe install/network/delete/Git-like actions.
- Pending requests do not execute after duplicate decision, cancellation, or expiry. Human denial returns a rejected ToolResult to the next LLM turn.
- Windows path case variants and PowerShell executable/flag spelling variants are covered by regression tests.

## Residual risk

The shell classifier intentionally does not implement a general shell parser. It rejects composition and known dynamic/encoded PowerShell forms, then requires HITL for the remaining simple free-shell commands. This is the specified T-004 boundary, not a claim that arbitrary shell text is safe to auto-execute.
