# T-004 Governance and HITL Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` task-by-task. Steps use checkbox syntax.

**Goal:** Put deterministic workspace governance and HITL approval in the only AgentRunner-to-Dispatcher path.

**Architecture:** Guardrail classifies each validated Action; ApprovalStore owns immutable requests and scoped grants; RunStateMachine validates status transitions. AgentRunner suspends a copied run state for approval and resumes only the stored action after a valid decision.

**Tech Stack:** TypeScript strict, Vitest, Zod contracts already present, injected FakeClock and fake dispatcher; no new dependency.
